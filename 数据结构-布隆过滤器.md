#### 布隆过滤器
```
介绍: 是一种数据结构，特点是高效插入和查询，用来判断"某数据一定不存在或者可能存在",
一般是通过redis来实现;
```
(1)原理
```
判断某个元素是否存在，一般使用hashMap, O(1)复杂度返回结果，但是当数据量很大，hashMap就过大而无法使用；

布隆过滤器是一个bit数组，每位是0或1；
要映射一个布隆过滤器，需使用多个不同且固定的hash函数生成数据的多个hash值,
这几个hash值指向位置的bit变为1；
例如： nihao的三个hash值为1, 4, 7； 
bit数组从0,0,0,0,0,0,0变成0,1,0,0,1,0,0,1
这样之后来判断nihao是否存在，如果这三个位置有0，那就一定不存在；
如果都是1，那就可能存在；

```
(2)如何选择hash函数个数与bit数组长度
```
如果bit数组太小，hash函数个数少，很容易就出现匹配情况，导致误报情况很多；
hash函数个数越大，bit数组越长其误报率越小；
需要根据数据量选择合适的值
```
(3)实践说明
```
实际应用，一般是用redis储存，其支持setBit, getBit,内存操作速度快；
生产情况容易出现较大的key，读写负担太大，需要进行拆分，变成多个key，组合完成布隆过滤器的功能；

```
